Overview

Purpose

The purpose of this site is to demonstrate a really cool piece of software I have been working on in my spare time and most of this last summer. I am not ready to launch or reveal publicly yet, but I want to have a site ready to go when I am.

Audience

The primary audience for this project will be people who like automation and AI—ideally, people who would like to free up their time by removing mundane tasks.

Dynamic elements

JavaScript will power an interactive “Idea Board” and a “Feature Spotlight” gallery. An ES module will export an array of idea objects (with fields such as title, type, description, and upvoteCount) and another array of example “demo features” that represent safe, high-level capabilities of the software. A main module will import this data and render it into cards on the page using DOM manipulation and template functions.

Visitors will be able to filter ideas by type (Feature, Bug, Integration, Other) and sort them by newest or highest upvotes using array methods like filter, sort, and map. Each idea card will include an “Upvote” button that increments the upvote count in the underlying object and re-renders the card. A simple form will let users add new ideas for this session only; submitted ideas will be pushed into the ideas array and displayed immediately. Conditional logic will show a friendly message if no ideas match the current filters and will highlight “Popular” ideas (for example, any item with more than 5 upvotes).

Branding

Website Logo

The final logo direction is a simple, professional, single-stroke robot mark that scales well. The icon will typically use the primary color, with the wordmark in the dark accent color (or white when placed on a dark background).

Content

Home page

The home page will introduce Really Cool Software at a high level without revealing all of its internal details. It will explain that this is a desktop assistant project focused on automation, but emphasize that the site is a preview and idea hub rather than a full product launch.

  • A hero section with the project name, a short tagline, and a brief paragraph.
  • A “Feature Spotlight” area that uses JavaScript to display a small set of safe, high-level example capabilities from a features array.
  • A simple overview of how visitors can participate: reading the roadmap, submitting ideas, and upvoting suggestions on the Idea Board.
  • At least one image that represents software, automation, or a clean desktop workspace, plus the site logo.

Idea Board (Page 2)

The Idea Board page will be the main interactive part of the site where visitors can view and interact with example ideas and feature suggestions related to Really Cool Software.

  • A short introduction explaining this is a preview space for collecting ideas.
  • Filters to select idea type (Feature, Bug, Integration, Other) and sorting by newest or highest upvotes using array methods like filter and sort.
  • A dynamically generated list of idea cards from an array of objects; clicking “Upvote” increases upvoteCount and updates the UI.
  • A form that adds new ideas for this session only and re-renders the list.
  • Conditional messages (e.g., “No ideas match your filters”) and a “Popular” highlight for ideas above a threshold.
  • At least one relevant image/icon that symbolizes brainstorming, collaboration, or feature requests.

Build Checklist (Course Requirements + Instructor Feedback)

Course Requirements

  • Site has a home page and at least one child page.
  • Site is hosted on the web.
  • Each page has significant textual content appropriate for the audience.
  • Each page includes at least one relevant image.
  • Working links to child page(s) exist on each page.
  • Pages follow the approved wireframes (structure and intent).
  • Responsive design (mobile to desktop).
  • Visually appealing and easily readable.
  • Valid HTML and CSS.
  • Lighthouse: 96+ Performance and 96+ Accessibility.
  • JavaScript includes:
    • More than one function (organized code).
    • DOM interaction (select, modify, event listener).
    • Conditional branching.
    • Objects, Arrays, and array methods (forEach/map/reduce/filter/sort).

Instructor Feedback Implementation

  • Logo: Use a simple, professional logo (single-stroke robot mark).
  • Color usage: Apply the palette consistently across header, buttons, section backgrounds, and cards.
  • Palette alignment: If the final UI uses different colors, update the palette to match the UI. Otherwise, enforce the palette in the UI.
  • Home layout: Improve hierarchy and spacing using a clear hero + spotlight + participation layout.

Brand Application Rules

  • Primary (#396E94): header, headings, borders, key accents.
  • Secondary (#B5983E): buttons, highlights, badges, interactive emphasis.
  • Accent 1 (#2B1C2C): body text, footer, deep contrast elements.
  • Accent 2 (#AAAABB): background panels, subtle dividers, muted surfaces.
  • Logo color use: icon in Primary, wordmark in Accent 1 (or white when on dark backgrounds).
